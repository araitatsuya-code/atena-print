# 06. プレビュー一致性 QA ガイド

Issue: [#47](https://github.com/araitatsuya-code/atena-print/issues/47)

## 目的

- プレビュー（Canvas）と印刷用出力（ラベル画像/PDF）の一致性を回帰テストで継続監視する
- 主要レイアウト（縦書き/横書き、A4 10面/12面）を実機印刷でも検証する

## 自動回帰テスト

### 対象ケース

- 縦書き A4 12面
- 横書き A4 12面
- 縦書き A4 10面
- 横書き A4 10面

### 実行コマンド

```bash
cd frontend
npx vitest run src/lib/labelSnapshot.parity.test.ts

cd ..
go test ./internal/infrastructure/pdf
```

### 検証内容

- `labelSnapshot.parity.test.ts`
  - 96dpi 基準で、プレビュー相当描画と印刷用スナップショット描画の描画命令列が一致すること
- `label_imposition_test.go`
  - A4 10面/12面の面付け座標とページ分割（満杯、+1件改ページ）が期待通りであること

## 実機印刷チェックリスト（100% 実寸）

### 事前条件

- プリンタドライバの拡大縮小を無効化（100% / 実際のサイズ）
- 用紙設定をラベル用紙実寸に合わせる（A4）
- アプリ内レイアウト設定を対象ケースに合わせる

### 実施ケース

1. 縦書き A4 12面
2. 横書き A4 12面
3. 縦書き A4 10面
4. 横書き A4 10面

### 各ケースの確認項目

- [ ] 1列目左上ラベルの基準位置が想定内（X/Y）
- [ ] 最終ラベル（12面: 12番、10面: 10番）が枠内に収まる
- [ ] 氏名/住所の改行位置がプレビューと一致する
- [ ] 郵便番号の桁間隔・ハイフン位置がプレビューと一致する
- [ ] QR コードが読取可能である（有効時）
- [ ] 透かし濃度が可読性を阻害しない（有効時）
- [ ] 11件目（10面）/13件目（12面）で2ページ目の先頭位置が正しい

### Issue 添付ルール

- 各ケースで以下を Issue #47 に添付する
  - 印刷結果（写真またはスキャン）
  - 使用した設定（向き、面数、オフセット、プリンタ設定）
  - 判定結果（OK/NG）と補正値

## 既知の誤差許容範囲

|項目|許容範囲|備考|
|---|---|---|
|プレビュー相当描画 vs スナップショット描画|0 差分（自動テストで完全一致）|描画命令列の一致で検証|
|面付け座標（計算値）|±0.1 mm|浮動小数点丸め誤差の許容|
|実機印刷の位置ずれ（X/Y）|±0.5 mm|用紙送り・機種差を考慮|
|QR 読取性|100% 読み取り可能|標準スマホカメラで確認|

## NG 時の対処

1. まずオフセット補正（`offsetX` / `offsetY`）で調整
2. 0.5mm を超える恒常ずれが続く場合はプリンタ別プロファイル化を検討
3. 自動テスト差分が出た場合は `labelRenderer.ts` / `labelSnapshot.ts` / `label_imposition_test.go` の変更点を優先確認
